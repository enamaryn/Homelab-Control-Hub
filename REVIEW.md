# Axes d'amélioration proposés

1. **Accessibilité et clavier** : Les modales (`id="modal"`, `settings-modal`, `camera-modal`) n'exposent pas d'attributs `role="dialog"`, `aria-modal` ou de gestion de focus/`aria-labelledby`. Ajouter ces attributs, gérer le focus initial et le piège de tabulation améliorerait l'expérience pour les lecteurs d'écran et la navigation clavier.
2. **Fiabilité du check de santé** : Le ping des services repose sur un `fetch` en mode `no-cors` vers l'URL du service, ce qui peut retourner des faux positifs/négatifs (les réponses sont opaques et l'état HTTP n'est pas lisible). Prévoir un endpoint de healthcheck dédié (HEAD/JSON) ou passer par un petit proxy côté LAN pour obtenir un statut HTTP réel.
3. **Organisation et performance** : Le fichier unique combine un gros bloc CSS et tout le JavaScript inline. Extraire ces parties dans des fichiers séparés (avec hash pour cache-busting) faciliterait la maintenance, activerait le cache navigateur et ouvrirait la porte à un bundler/minification.
4. **Stockage des logos/images** : Les logos/imports d'icônes sont encodés en base64 et stockés dans `localStorage`, qui a une capacité limitée (~5–10 Mo selon le navigateur). Avec des images de 256 Ko par entrée, la saturation peut arriver vite. Stocker des URL externes, des versions optimisées (WebP) ou utiliser `IndexedDB` pour les blobs rendrait la solution plus robuste.
